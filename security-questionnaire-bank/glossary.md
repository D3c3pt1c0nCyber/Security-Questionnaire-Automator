# Security Glossary & Standard Terms

Use these definitions consistently across all questionnaire answers.
Claude Code should reference this glossary to ensure terminology is accurate and consistent.

## Compliance & Certifications
| Term | Definition |
|------|-----------|
| **SOC 2 Type II** | Service Organization Control report auditing security, availability, processing integrity, confidentiality, and privacy over a period of time |
| **ISO 27001** | International standard for Information Security Management Systems (ISMS) |
| **PCI DSS** | Payment Card Industry Data Security Standard for handling cardholder data |
| **HIPAA** | Health Insurance Portability and Accountability Act — US healthcare data protection |
| **FERPA** | Family Educational Rights and Privacy Act — US student education records |
| **GDPR** | General Data Protection Regulation — EU data privacy law |
| **CCPA/CPRA** | California Consumer Privacy Act / California Privacy Rights Act |
| **FedRAMP** | Federal Risk and Authorization Management Program — US government cloud security |
| **StateRAMP** | State Risk and Authorization Management Program — state/local government equivalent |
| **CJIS** | Criminal Justice Information Services — FBI security policy for law enforcement data |
| **NIST CSF** | National Institute of Standards and Technology Cybersecurity Framework |
| **CIS** | Center for Internet Security — provides security benchmarks and controls |

## Security Concepts
| Term | Definition |
|------|-----------|
| **MFA** | Multi-Factor Authentication — requires 2+ verification methods |
| **SSO** | Single Sign-On — one set of credentials for multiple applications |
| **SAML** | Security Assertion Markup Language — XML-based SSO protocol |
| **OIDC** | OpenID Connect — identity layer on top of OAuth 2.0 |
| **RBAC** | Role-Based Access Control — permissions assigned by role |
| **PAM** | Privileged Access Management — controls for admin/elevated accounts |
| **DLP** | Data Loss Prevention — tools to prevent unauthorized data exfiltration |
| **EDR** | Endpoint Detection and Response — advanced endpoint security |
| **SIEM** | Security Information and Event Management — centralized log analysis |
| **SOC** | Security Operations Center — team monitoring security 24/7 |
| **WAF** | Web Application Firewall — protects web apps from attacks |
| **IDS/IPS** | Intrusion Detection/Prevention System |
| **ZTNA** | Zero Trust Network Access — "never trust, always verify" model |

## Encryption
| Term | Definition |
|------|-----------|
| **AES-256** | Advanced Encryption Standard with 256-bit key — industry standard for data at rest |
| **TLS 1.2/1.3** | Transport Layer Security — encrypts data in transit |
| **KMS** | Key Management Service — manages encryption keys |
| **HSM** | Hardware Security Module — dedicated hardware for key protection |
| **BYOK** | Bring Your Own Key — customer-managed encryption keys |

## Development & Operations
| Term | Definition |
|------|-----------|
| **SDLC** | Software Development Life Cycle |
| **SAST** | Static Application Security Testing — scans source code |
| **DAST** | Dynamic Application Security Testing — scans running applications |
| **SCA** | Software Composition Analysis — scans dependencies for vulnerabilities |
| **SBOM** | Software Bill of Materials — inventory of software components |
| **CI/CD** | Continuous Integration / Continuous Deployment |
| **SLSA** | Supply-chain Levels for Software Artifacts — build integrity framework |

## Risk & Recovery
| Term | Definition |
|------|-----------|
| **RTO** | Recovery Time Objective — max acceptable downtime |
| **RPO** | Recovery Point Objective — max acceptable data loss (time) |
| **BCP** | Business Continuity Plan |
| **DRP** | Disaster Recovery Plan |
| **TPRM** | Third-Party Risk Management |
| **CVSS** | Common Vulnerability Scoring System — rates vulnerability severity (0-10) |

## Privacy
| Term | Definition |
|------|-----------|
| **DPO** | Data Protection Officer |
| **DSAR** | Data Subject Access Request |
| **DPA** | Data Processing Agreement |
| **PIA/DPIA** | Privacy Impact Assessment / Data Protection Impact Assessment |
| **PII** | Personally Identifiable Information |
| **SCCs** | Standard Contractual Clauses — legal mechanism for cross-border data transfers |

## Questionnaire Frameworks
| Term | Definition |
|------|-----------|
| **HECVAT** | Higher Education Community Vendor Assessment Toolkit (by EDUCAUSE) |
| **SIG** | Standardized Information Gathering questionnaire (by Shared Assessments) |
